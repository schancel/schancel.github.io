X Thread: Getting Started with Finch

[1/8] If you have X Premium+, you already have free access to Grok’s API.

Here’s how to turn that into a coding assistant that reads your files, runs your tests, and remembers what you worked on last week. 🧵

[2/8] First, install iTerm2. It’s a better terminal for Mac and Finch looks great in it.

→ iterm2.com → Download → drag to Applications

Then open it. You’ll see a blinking cursor. That’s where we’re working.

[3/8] Get your Grok API key:

→ console.x.ai → Sign in with your X account → API Keys → Create API Key → Copy it (starts with xai-)

Save it somewhere — it’s only shown once.

[4/8] Install Finch. Paste this in iTerm2:

Apple Silicon (M1/M2/M3/M4):

curl -L github.com/schancel/shammah/releases/latest/download/finch-macos-arm64.tar.gz | tar xz && sudo mv finch /usr/local/bin/

Intel Mac:

curl -L github.com/schancel/shammah/releases/latest/download/finch-macos-x86_64.tar.gz | tar xz && sudo mv finch /usr/local/bin/

[5/8] Configure it:

finch setup

Arrow keys to navigate. Press A to add a provider, pick Grok, paste your key. Press F to save.

Takes 30 seconds.

[6/8] Start it:

finch --cloud-only

No model download. No daemon. Just you and Grok in your terminal with a full tool loop.

Try:

> Read my README.md and suggest improvements

It will ask before touching anything.

[7/8] The part I actually use it for:

Navigate to your project folder first:

cd ~/my-project
finch --cloud-only

Now ask:

> Run the tests and explain what's failing
> Find everywhere I'm using the deprecated API
> What did we change in the auth flow last week?

That last one works because Finch stores conversation history locally and searches it semantically.

[8/8] Everything stays on your machine. No account. No cloud storage.

The only traffic is your message → Grok → response back.
